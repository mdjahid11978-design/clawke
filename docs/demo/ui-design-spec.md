# 技能中心 UI 设计规范

## 适用范围

本文档记录技能中心页面本轮确认的 UI 规范，当前对应预览文件：

- `docs/demo/skill-center-ui-demo.html`

## 核心原则

技能中心是管理型列表页面，不是展示页。设计优先级按以下顺序：

1. 扫描效率
2. 信息稳定
3. 操作明确
4. 响应式适配
5. 视觉一致性

## 布局规范

整体结构保持稳定：

- 左侧主导航
- Gateway 列表
- 技能中心主内容区

局部 UI 优化不得随意改动外层结构。技能卡片内部可以根据内容和宽度做响应式调整。

## 技能卡片规范

技能卡片采用最小宽度驱动布局：

- 内容区宽度足够时，卡片可显示为两列。
- 内容区宽度不足时，卡片自动退回一列。
- 移动端保持一列。
- 最多两列，不做三列或更多列。

卡片主要信息包括：

- 技能名称
- 来源标签
- 技能类型标签
- 技能描述
- 文件路径
- 启用状态
- 编辑 / 删除操作

## 标签规范

标签只保留高价值信息：

- 来源标签，例如 `openclaw-bundled`、`agents-skills-personal`
- 类型标签，例如 `内置技能`、`外部技能`、`只读来源`

避免重复统计和低价值装饰信息，减少视觉噪音。

## 描述规范

列表页技能描述最多显示 3 行。

原因：

- 保持卡片高度稳定。
- 避免长描述挤压操作区。
- 保证两列布局下的扫描效率。

详情页可以显示完整描述。

## 控制区规范

控制区包括：

- 启用 / 禁用状态
- 启用 / 禁用开关
- 编辑按钮
- 删除按钮

响应式规则：

- 宽度足够时，控制区位于卡片右侧，竖向排列。
- 宽度不足时，控制区移动到卡片底部，横向一行展示。
- 底部横向展示时整体靠右对齐。

## 移动端规范

移动端保持信息优先：

- 技能卡片一列展示。
- 控制区在卡片底部横向排列。
- 搜索和筛选控件可换行。
- Gateway 列表可压缩为横向或上方区域。

## 实施建议

Flutter 实现时建议使用：

- `LayoutBuilder` 判断主内容区宽度。
- `GridView` 或 `Wrap` 根据最小卡片宽度决定一列 / 两列。
- 卡片内部再用 `LayoutBuilder` 判断控制区是否放右侧或下方。
- 技能描述使用 `maxLines: 3` 和 `TextOverflow.ellipsis`。

## 当前结论

选定版本：

- 方案 C 原始小卡片布局。
- 卡片按宽度自动一列 / 两列。
- 描述显示 3 行。
- 控制区宽度足够时在右侧，宽度不足时移动到底部并靠右一行显示。
